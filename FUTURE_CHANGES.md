# Будущие изменения Realmcraft

Дата актуализации: 23 июля 2026 года.

Этот файл — рабочий список изменений, которые всё ещё нужны для сближения
Realmcraft с Minecraft Java Edition 1.2.x. Он дополняет `DIFF.md`, `ROADMAP.md`
и `BUGS.md`, но не заменяет их.

Обозначения:

- `[ ]` — не сделано;
- `[-]` — реализовано частично;
- `[x]` — реализовано и проверено тестами.

## 0. Зафиксировать точную целевую версию

[-] В документации основной целью названа Minecraft Java Edition 1.2.4, а
локальный исходный референс и новый порт генерации относятся к 1.2.5.

Нужно:

- [ ] выбрать единственную эталонную версию для механик и генерации;
- [ ] подписать версию рядом с каждым намеренным отличием;
- [ ] не переносить более новые механики без явной отметки;
- [ ] добавить короткий список допустимых отличий Realmcraft от эталона.

До принятия другого решения в этом документе под «vanilla» подразумевается
Minecraft Java Edition 1.2.4, а код 1.2.5 используется как близкий технический
референс для генерации мира.

## 1. Состояние сущностей и сохранения

Перед расширением AI стоит добавить недостающие поля в единое состояние
сущности и в сериализацию.

- [ ] `woolColor` для овцы;
- [ ] `tamed`, `ownerId`, `sitting` и отдельное состояние гнева для wolf/cat;
- [ ] вариант кошки для сохранения, а не вычисление только из ID сущности;
- [ ] `charged` для creeper;
- [ ] запас воздуха для водных существ;
- [ ] экипировка мобов, как минимум головной слот;
- [ ] необходимые таймеры AI без использования одного поля для разных задач;
- [ ] безопасные значения по умолчанию при загрузке старого сохранения;
- [ ] round-trip тесты для каждого нового поля.

Старые чанки мира мигрировать не требуется: проект использует только текущую
версию генерации. Однако состояние уже существующих сущностей должно
загружаться без ошибки.

## 2. Категории появления существ

Сейчас squid, wolf, ocelot, cat и golem входят в общую категорию peaceful и
делят один passive cap.

- [ ] выделить категории `creature`, `waterCreature` и `ambient`;
- [ ] установить отдельные caps, включая water 5 и ambient 15;
- [ ] считать caps по eligible chunks;
- [ ] не учитывать созданных игроком и приручённых существ как обычный
  результат natural spawn там, где это соответствует vanilla;
- [ ] исключить golem из обычных таблиц размножения и natural passive spawn;
- [ ] проверять pack limit и правила конкретного вида для каждого участника;
- [ ] уточнить эталонные биомы и диапазон высоты появления squid.

### Дополнительно замечено

Текущая общая функция `feed()` воспринимает кость для wolf и рыбу для ocelot
как предметы обычного размножения. До появления отдельных состояний приручения
это создаёт неверную механику.

- [ ] разделить `tame`, `heal`, `breed` и `tempt`;
- [ ] запретить универсальному `breedPairs()` размножать неподходящие peaceful
  виды;
- [ ] покрыть это отрицательными тестами для squid, golem и неприручённых
  wolf/ocelot.

## 12. Генерация мира: оставшаяся точность

[-] Java Random, GenLayer, octave density, поверхности, вода/лёд,
chunk-crossing caves, ravines и lakes уже перенесены в текущую генерацию.

Полная поблочная совместимость пока не доказана и не охватывает весь population
pipeline.

- [ ] сделать differential harness против настоящего Java-генератора;
- [ ] сравнивать biome IDs, height map и полный массив блоков выбранных чанков;
- [ ] добавить эталонные snapshots для положительных, отрицательных и строковых
  seed;
- [ ] проверить точный порядок consumption Java Random между этапами;
- [ ] перенести структуры на точные Java placement/start/component rules;
- [ ] сверить villages, mineshafts, strongholds и dungeons блок за блоком;
- [ ] перенести точные biome decorators и генераторы деревьев;
- [ ] сверить руды, растения, снег и замерзание воды;
- [ ] проверить population на границах чанков и независимость от порядка
  загрузки;
- [ ] определить поведение пустой строки seed: случайный seed интерфейса
  vanilla или детерминированный `0`.

Критерий готовности: набор эталонных seed и координат даёт одинаковые блоки с
Java-референсом либо каждое разрешённое отличие явно перечислено.

## 14. Общие критерии готовности

Для каждого завершённого пункта:

- [ ] состояние сохраняется и восстанавливается;
- [ ] natural spawn, structure spawn и ручной spawn используют общую
  валидацию;
- [ ] механика не использует более новую версию Minecraft без явной отметки;
- [ ] есть unit-тест основного правила и хотя бы один отрицательный тест;
- [ ] есть интеграционный тест взаимодействия с миром;
- [ ] исходные `.ts` файлы остаются не длиннее 600–700 строк;
- [ ] проходят `npm test` и `npm run build`;
- [ ] `DIFF.md`, `ROADMAP.md` и этот файл обновлены одновременно.
