# REALMCRAFT

Realmcraft — любительский браузерный voxel-sandbox в стиле Minecraft Java Edition
1.2.4. Проект написан на TypeScript и Three.js, собирается через Vite и работает
непосредственно в браузере через WebGL.

В игре уже есть процедурный бесконечный мир, биомы и пещеры, руды, режимы
Survival и Creative, здоровье и голод, инвентарь, крафт 2×2 и верстак 3×3,
печь с переплавкой и топливом, сундуки (включая двойные), инструменты с
прочностью и уровнями добычи, физические дропы, сохранение мира, погода, смена
времени суток и классический интерфейс Minecraft 1.2.4.

> Это не официальный продукт Mojang/Microsoft. Оригинальные ассеты Minecraft
> предназначены только для локального личного использования и не должны
> распространяться вместе с публичной сборкой проекта.

## Быстрый запуск

Требуются Node.js и npm. В текущей рабочей копии классические ассеты уже лежат в
`public/assets/minecraft`.

```bash
npm install
npm run dev
```

Vite выведет локальный адрес, обычно `http://localhost:5173`. На стартовом экране
нужно выбрать seed, качество графики и режим игры, нажать **Enter World**, а затем
щёлкнуть по игровому окну для захвата курсора.

Production-сборка и её локальный просмотр:

```bash
npm run build
npm run preview
```

### Команды npm

| Команда | Назначение |
| --- | --- |
| `npm run dev` | Запустить Vite в режиме разработки |
| `npm run build` | Проверить TypeScript и собрать production-версию в `dist/` |
| `npm run preview` | Локально открыть production-сборку |
| `npm run textures:install` | Старый вспомогательный скрипт для современных текстур 26.2; текущий классический atlas 1.2.4 его не использует |

## Управление

| Клавиша | Действие |
| --- | --- |
| `W A S D` | Передвижение |
| Мышь | Поворот камеры |
| `Space` | Прыжок, всплытие или полёт вверх |
| `Shift` | Спринт |
| `C` или `Ctrl` | Присесть или лететь вниз |
| ЛКМ, удерживать | Ломать блок |
| ПКМ | Поставить выбранный блок; по верстаку, печи или сундуку — открыть их окно |
| Колесо мыши / `1`–`9` | Выбрать слот хотбара |
| `E` | Открыть или закрыть инвентарь в Survival |
| `O` | Временная админ-панель: все предметы, ЛКМ +1, ПКМ +стак |
| `Q` | Выбросить один предмет из выбранного стака |
| `F` | Включить фонарик |
| `G` | Включить полёт в Creative |
| `R` | Переключить страницу Creative-хотбара |
| `X` | Включить режим проверки руд: xray + noclip |
| `T` | Ускорить течение игрового времени |
| `Y` | Переключить погоду: clear → cloudy → rain → storm |
| `Esc` | Пауза и освобождение курсора |

## Режимы игры

### Survival

- 20 HP, 20 единиц голода и отдельный запас воздуха под водой.
- Урон от падения, утопления и полного истощения голода.
- При голоде не меньше 18 здоровье постепенно восстанавливается, увеличивая
  истощение игрока.
- Спринт, плавание, прыжки, разрушение и установка блоков расходуют энергию.
- После смерти игрок возрождается с восстановленными здоровьем и голодом.
- Разрушенные блоки выпадают физическими предметами и подбираются при приближении;
  близкие одинаковые дропы объединяются, число предметов-сущностей ограничено.
- Инвентарь содержит 36 слотов, первые 9 являются хотбаром; размер стака — 64,
  инструменты не стакаются.
- Установка блока расходует один предмет из выбранного стака.

#### Крафт и инструменты

- В инвентаре доступна личная сетка крафта 2×2; верстак открывает сетку 3×3
  (ПКМ по установленному верстаку; с присяданием блок ставится поверх него —
  то же правило действует для печи и сундука).
- ЛКМ по слоту берёт или кладёт весь стек, ПКМ берёт половину или кладёт один
  предмет; при закрытии окна содержимое сетки возвращается в инвентарь.
- Рецепты: доски из брёвен, палки, верстак, печь, сундук, факелы и полный набор
  инструментов — кирки, топоры, лопаты, мотыги и мечи из дерева, камня, железа,
  золота и алмаза.
- В верстаке есть книга рецептов (кнопка-книга в правом верхнем углу окна):
  панель показывает все рецепты, недоступные по материалам затемняются,
  клик по доступному автоматически раскладывает ингредиенты из инвентаря по сетке. Готовой
  текстуры recipe book в ассетах 1.2.4 нет (механика появилась в 1.12), поэтому
  панель собрана из классической палитры GUI и иконки книги из `gui/items.png`.
- Инструменты ускоряют добычу подходящих блоков, имеют прочность и ломаются;
  износ отображается полоской в слоте и сохраняется в мире.
- Уровни добычи: каменные блоки требуют кирку, железная руда — каменную кирку,
  золотая и алмазная руды — железную. Без подходящего инструмента блок ломается
  медленно и не даёт дроп.
- Правильные дропы: камень → булыжник, трава → земля, угольная руда → уголь,
  алмазная руда → алмаз, листья и стекло не выпадают; железная и золотая руды
  падают блоками и переплавляются в печи.

#### Печь и сундуки

- ПКМ по печи открывает классическое окно с тремя слотами (сырьё, топливо,
  результат), индикатором пламени и стрелкой прогресса; каждый предмет
  переплавляется 10 секунд.
- Рецепты переплавки: железная и золотая руда → слитки, песок → стекло,
  булыжник → камень. Переплавка еды появится вместе с едой в этапе 6.
- Топливо горит разное время: уголь 80 с, доски и брёвна 15 с, деревянные
  инструменты 10 с, палки 5 с.
- Печь продолжает работать при закрытом окне и после перезагрузки страницы;
  поставленная печь разворачивается к игроку, а горящая передняя грань светится и
  слегка мерцает.
- Сундук хранит 27 слотов. Два сундука, поставленные вплотную, образуют двойной
  сундук на 54 слота с единой большой моделью; сундуки разворачиваются к игроку,
  а рядом с двойным сундуком новый сундук поставить нельзя.
- При разрушении печи или сундука всё содержимое выпадает предметами.

Цветы, высокая трава, факелы и другие cross-элементы отображаются плоскими
прозрачными спрайтами в инвентаре, руке и среди выпавших предметов. Для них не
рисуется кубическая рамка наведения.

### Creative

- Урон, голод и конечные стаки отключены.
- Доступен бесконечный набор блоков на нескольких страницах хотбара.
- `R` переключает страницы, `G` включает обычный Creative-полёт.

### Xray / noclip

`X` включает служебный режим осмотра генерации руд. Обычные блоки становятся
прозрачными, руды остаются видимыми, столкновения и гравитация отключаются. Режим
доступен для тестирования и в Creative, и в Survival; в Survival во время noclip
не расходуется голод и не применяется урон от окружения.

## HUD и интерфейс

Интерфейс повторяет логические размеры и pixel-art стиль Minecraft 1.2.4:

- bitmap-шрифт загружается из `font/default.png`;
- масштаб GUI выбирается целым множителем с учётом размера окна;
- хотбар и рамка выбранного слота используют `gui/gui.png`;
- сердца, голод, воздух и полоса опыта используют `gui/icons.png`;
- окно Survival-инвентаря использует `gui/inventory.png`;
- иконки блоков строятся из тех же atlas-тайлов, что и блоки в мире;
- при получении урона появляются красная вспышка, покачивание камеры, анимация
  сердец и классический звук боли.

Полоса опыта уже присутствует в HUD, но пока всегда пустая: уровни, XP-сферы и
логика заполнения будут добавлены позднее.

В левом верхнем углу выводятся FPS, координаты, биом, игровое время, погода и seed.
Все строки используют один размер bitmap-шрифта.

## Мир и генерация

Мир детерминирован seed-строкой и генерируется чанками. Один и тот же seed создаёт
тот же базовый ландшафт; пользовательские изменения сохраняются отдельно.

Реализованы:

- ocean, beach, plains, forest, desert, mountains, snowfields и river;
- холмы, берега, океаны, реки и системы пещер;
- дубы и ели, высокая трава и цветы;
- вода, плавание и подводный визуальный эффект;
- падающие песок и гравий;
- угольная, железная, золотая и алмазная руда.

### Высоты и частота руд

| Руда | Высота Y | Попыток жил на чанк | Размер жилы |
| --- | ---: | ---: | ---: |
| Уголь | 5–96 | 10 | 8–16 |
| Железо | 5–64 | 8 | 5–10 |
| Золото | 5–32 | 2 | 4–7 |
| Алмазы | 5–18 | 1 | 3–6 |

Жилы генерируются детерминированно после пещер и заменяют только камень, поэтому
часть руды естественным образом видна на стенах пещер.

## Сохранения

Состояние хранится в браузерном `localStorage`, отдельно для каждого seed.

- Ключ мира: `realmcraft.world.v1.<encoded-seed>`.
- Ключ настроек: `realmcraft.settings.v1`.
- Автосохранение выполняется каждые 8 секунд, при паузе, скрытии/закрытии страницы
  и выходе в главное меню.
- Сохраняются позиция и направление игрока, HP, голод, воздух, инвентарь, лежащие
  в мире предметы, содержимое сундуков и печей вместе с прогрессом переплавки,
  выбранный слот/страница, время суток, погода, noclip и все изменённые игроком
  блоки, включая направление печей и сундуков.
- Качество, громкость, head bob, последний seed и последний выбранный режим
  сохраняются как настройки.

Очистка данных сайта в браузере удалит сохранённые миры.

## Классические ассеты Minecraft 1.2.4

Текущий renderer использует старый формат ресурсов до Minecraft 1.5.

### Текстуры

Главный источник блоков — `public/assets/minecraft/terrain.png`. Во время запуска
`src/gfx/Atlas.ts` выбирает из него необходимые тайлы и собирает компактный runtime
atlas 8×8 с размером тайла 16×16. Там же создаются стадии разрушения блоков и
цветовые варианты травы/листвы.

Файлы, которые непосредственно загружает текущий код:

```text
public/assets/minecraft/
  terrain.png
  font/default.png
  gui/gui.png
  gui/icons.png
  gui/inventory.png
  gui/crafting.png
  gui/furnace.png
  gui/container.png
  gui/items.png
  sound/...
```

Остальные каталоги старого клиента (`mob`, `armor`, `art`, `title`, `item` и
другие) сохранены как исходный набор для будущих этапов, но сейчас напрямую не
используются.

### Звуки

В локальном старом client JAR звуки не находились: лаунчер Minecraft загружал их
отдельно в папку `%APPDATA%\.minecraft\resources`. Поэтому звуки были восстановлены
через официальный Mojang asset index `pre-1.6`:

- [официальный индекс pre-1.6](https://launchermeta.mojang.com/v1/packages/3d8e55480977e32acd9844e545177e69a52f594b/pre-1.6.json);
- файлы загружаются с `https://resources.download.minecraft.net/<2 символа SHA-1>/<SHA-1>`;
- локальная папка проекта: `public/assets/minecraft/sound`.

В проекте находятся 44 оригинальных OGG-файла:

```text
sound/
  step/               cloth, grass, gravel, sand, snow, stone, wood (по 4)
  damage/             fallbig, fallsmall, hurtflesh
  liquid/             splash
  random/             click, pop
  ambient/weather/    rain, thunder
```

`src/audio/Audio.ts` асинхронно декодирует их через Web Audio API. Оригинальные
семплы используются для шагов, разрушения/установки блоков, падения, получения
урона, воды, подбора предметов, дождя и грома. Если конкретный файл не загрузился,
остаётся синтезированный fallback. Ветер и птицы всегда генерируются кодом, потому
что отдельных подходящих семплов в используемом классическом наборе нет.

## Графика и погода

- Three.js/WebGL, nearest filtering и pixel-art текстуры.
- Greedy meshing, отсечение невидимых граней и frustum culling.
- Ambient occlusion на вершинах и динамические тени.
- ACES tone mapping, дистанционный туман и атмосферное небо.
- Солнце, луна, звёзды, процедурные облака и смена дня/ночи.
- Анимированная вода с отражённым бликом и подводной цветокоррекцией.
- Ветер для травы/листвы, дождь, снег, молнии, светлячки и декоративные птицы.

### Профили качества

| Профиль | Дальность, чанки | Shadow map | Pixel ratio cap | Плотность травы |
| --- | ---: | ---: | ---: | ---: |
| Low | 4 | 1024 | 1.0 | 35% |
| Medium | 6 | 2048 | 1.25 | 60% |
| High | 7 | 2048 | 1.5 | 100% |
| Ultra | 10 | 4096 | 2.0 | 100% |

Если FPS остаётся ниже 25 больше восьми секунд, игра автоматически уменьшает
дальность прорисовки.

Для вывода метрик производительности можно открыть:

```text
http://localhost:5173/?perf=1
```

Раз в две секунды в консоль выводятся FPS, CPU frame time, draw calls, triangles,
число загруженных чанков и дополнительные runtime-метрики.

```text
  install-minecraft-textures.sh  Старый installer современных block-текстур
```

## Текущие ограничения

- Полоса опыта декоративная; уровни и XP пока не реализованы.
- Нет еды и механики восстановления голода предметами; рецепты переплавки еды
  появятся вместе с самой едой.
- Нет враждебных мобов, боя и брони; мечи и мотыги уже крафтятся, но пригодятся
  в следующих этапах.
- Survival пока является фундаментом для следующих игровых систем, а не полной
  реализацией оригинального Minecraft.
